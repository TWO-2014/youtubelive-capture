# ショップチャンネル YouTube Live 分析パイプライン — 全体設計書

## 1. プロジェクト概要

### 目的

ショップチャンネル（通販番組）のYouTubeライブ配信を素材として、以下を解明する。

- **セールストークの話術構造** — なぜ視聴者が「買いたい」と思うのか、トークの型・パターンを抽出
- **番組構成・CV導線の設計** — 商品紹介からコンバージョン（購入）に至るまでの番組演出・構成の仕組み

### コンセプト

指定したYouTubeライブURLと時間を入力すると、音声文字起こし・画面キャプチャ・AI構造分析を自動実行し、レポートを出力するワンコマンドCLIツール。

## 3. 各ステップ詳細

### ① 入力（トリガー）

- **方式**: ユーザーがCLIでURL・録画時間を手動指定
- **インターフェース例**:
  ```bash
  python analyze.py \
    --url "https://www.youtube.com/watch?v=XXXXX" \
    --duration 60 \           # 分単位
    --screenshot-interval 30  # 秒単位（デフォルト30秒）
    --output ./reports/2026-04-01/
  ```
- **入力パラメータ**:
  | パラメータ | 必須 | デフォルト | 説明 |
  |---|---|---|---|
  | `--url` | ✅ | — | YouTubeライブ配信URL |
  | `--duration` | ✅ | — | 録画時間（分） |
  | `--screenshot-interval` | — | 30 | スクリーンショット間隔（秒） |
  | `--output` | — | `./output/{timestamp}/` | 出力ディレクトリ |
  | `--whisper-model` | — | `large-v3` | Whisperモデルサイズ |
  | `--skip-analysis` | — | false | 文字起こしまでで止める |

### ② 音声取得 + 定点スクリーンショット

#### 音声取得
- **ツール**: `yt-dlp`
- **処理**: ライブ配信（またはアーカイブ）から音声のみを抽出
- **出力**: `audio.wav`（Whisper入力用、16kHz mono）
- **コマンド例**:
  ```bash
  yt-dlp -x --audio-format wav \
    --postprocessor-args "-ar 16000 -ac 1" \
    --download-sections "*0-3600" \  # 最初の60分
    -o "audio.%(ext)s" \
    "<URL>"
  ```

#### 定点スクリーンショット
- **ツール**: `yt-dlp`（映像DL） + `ffmpeg`（フレーム抽出）
- **処理**: 指定間隔で画面キャプチャを取得
- **出力**: `screenshots/frame_000001.jpg`, `frame_000002.jpg`, ...（タイムスタンプをファイル名に含む）
- **コマンド例**:
  ```bash
  # 映像を取得しつつ、ffmpegで30秒おきにフレーム抽出
  yt-dlp -f "bestvideo[height<=720]" \
    --download-sections "*0-3600" \
    -o - "<URL>" | \
  ffmpeg -i pipe:0 -vf "fps=1/30" \
    -frame_pts 1 \
    screenshots/frame_%06d.jpg
  ```
- **最適化**: 解像度は720p以下で十分（テロップが読めればOK）。ファイル名にタイムスタンプ（秒）を埋め込み、後工程で音声と同期する。

#### 出力ディレクトリ構造
```
output/2026-04-01_1400/
├── audio.wav
├── screenshots/
│   ├── frame_0000_0s.jpg
│   ├── frame_0001_30s.jpg
│   ├── frame_0002_60s.jpg
│   └── ...
├── transcript.json
├── analysis.md
└── metadata.json
```

### ③ 音声文字起こし

- **ツール**: OpenAI Whisper（ローカル実行 or Whisper API）
- **モデル**: `large-v3`（日本語精度重視）
- **出力フォーマット**: タイムスタンプ付きJSON
  ```json
  {
    "segments": [
      {
        "start": 0.0,
        "end": 5.2,
        "text": "さあ、本日ご紹介するのはこちらの商品です"
      },
      {
        "start": 5.2,
        "end": 12.8,
        "text": "もう皆さんお待ちかねですよね"
      }
    ]
  }
  ```
- **VPSでのローカル実行の場合**: GPU無しだとlarge-v3は重い。`medium`で妥協するか、Whisper APIを使う判断が必要。
- **Whisper API利用時のコスト**: $0.006/分 → 60分で約$0.36（約55円）

### ④ 構造分析（Claude API マルチモーダル）

分析の核。文字起こしテキストとスクリーンショットをClaude APIに投げ、以下の観点で分析する。

#### 分析観点A: セールストークの話術構造

| 分析項目 | 内容 |
|---|---|
| トーク構造パターン | 問題提起 → 共感 → 解決策提示 → 実演 → 限定感 → CTA のような流れの抽出 |
| キーフレーズ抽出 | 繰り返し使われる訴求ワード（「今だけ」「特別価格」「残りわずか」等） |
| 話速・テンション変化 | 価格発表前後でのトーン変化、間（ま）の使い方 |
| 社会的証明の使い方 | 「○○個売れました」「お電話殺到」等のソーシャルプルーフ |
| 比較・アンカリング | 通常価格との比較、他社製品との差別化トーク |

#### 分析観点B: 番組構成・CV導線

| 分析項目 | 内容 |
|---|---|
| 時間配分 | 商品紹介・デモ・価格発表・クロージングの各フェーズの時間配分 |
| テロップ・画面演出 | 価格表示、カウントダウン、「残りわずか」演出のタイミングと内容 |
| CTAの設計 | 電話番号表示、QRコード表示のタイミングと頻度 |
| 繰り返し構造 | 同一商品内でのトーク→デモ→価格のループ回数とバリエーション |
| 商品切り替えの演出 | 商品間のトランジション、次商品への期待感の作り方 |

#### 実装方針

- 文字起こし全文を一括で投げるのではなく、**5〜10分のチャンク**に分割して分析
- 各チャンクに対応するスクリーンショット（3〜5枚）を同時にマルチモーダルで投入
- チャンクごとの分析結果を最後に統合し、番組全体の構造レポートを生成
- **2段階分析**: ① チャンク単位の詳細分析 → ② 全体統合・パターン抽出

#### Claude API コスト見積もり

- 60分の放送 → 約6〜12チャンク
- 各チャンク: テキスト約2,000トークン + 画像3〜5枚（各約1,500トークン）
- チャンクあたり入力: 約10,000トークン / 出力: 約2,000トークン
- 統合分析: 入力約15,000トークン / 出力約3,000トークン
- **合計概算**: 入力約135,000トークン + 出力約27,000トークン
- **Sonnet利用時**: 約$0.80〜$1.20 / 回

（話術の流れと特徴）

#### 出力先オプション

## 7. 利用上の注意

- YouTubeの利用規約上、映像・音声のダウンロードはグレーゾーン。分析目的の私的利用に留め、取得した映像・音声の再配布は行わない。
- 分析レポート自体は独自の著作物であり、公開・活用に制約はない。
- ショップチャンネルが YouTube ライブ配信を停止した場合、テレビ放送のHDMIキャプチャ方式への切り替えを検討する。
